# Гайд по настройке клиента Turnable &nbsp;·&nbsp; [🇬🇧 EN](SETUP.md)
## Быстрый старт
Напоминаем, что Turnable - это утилита для туннелирования, вам всё равно нужен VPN или proxy клиент. Мы рекомендуем [WireGuard](https://www.wireguard.com/quickstart/).

## Гайды для конкретной платформы
- [Настройка на Android](ANDROID_RU.md)
- [Настройка на Windows](WINDOWS_RU.md)
- [Настройка на Linux](LINUX_RU.md)

### 1. Получи конфигурацию
Попроси конфиг у оператора сервера или сгенерируй сам. Ты получишь JSON файл или URL вида turnable://.

### 2. Запусти клиент
```bash
./turnable client -l 127.0.0.1:1080 <config-файл-или-url>
```

Клиент будет слушать на адресе, который ты указал (по умолчанию `127.0.0.1:1080`).

### 3. Подключи своё приложение
Направь VPN или proxy клиент на адрес, где слушает Turnable. Готово!

## Флаги команды
```
-c, --config string    путь к JSON конфигу клиента (по умолчанию "config.json")
-l, --listen string    локальный TCP/UDP адрес для прослушивания (по умолчанию "127.0.0.1:0")
-i, --no-interactive   выключить интерактивный режим
-V, --verbose          включить подробное debug логирование
```

Можно указать либо путь к JSON файлу, либо URL конфигурации.

## Прямой relay к UDP серверу
Если хочешь подключиться прямо к серверу без промежуточного сервера Turnable, можешь сгенерировать прямой конфиг:

```bash
./turnable config direct <platform-id> <call-id> <username> <gateway-addr> -n [peers]
```

Флаги:
```
-n, --peers int   сколько peer соединений использовать (по умолчанию 1)
-j, --json        вывести конфиг в формате JSON
```

> [!WARNING]
> Это небезопасно и не рекомендуется. Используй только если знаешь, что делаешь.
